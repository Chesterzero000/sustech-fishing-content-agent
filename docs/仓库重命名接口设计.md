# 仓库重命名接口设计

## 目标

设计一个安全、可预览、可审计的仓库重命名接口，用于把当前 GitHub 仓库名调整为更清晰的英文 slug，同时保留中文项目名作为 README 和文档中的展示名称。

## 当前项目示例

- 当前仓库名：`fishing-agent-robot`
- 中文项目名：`南科大钓鱼协会内容机器人`
- 建议英文 slug：`sustech-fishing-content-agent`

## 接口总览

```text
POST /api/github/repositories/rename/preview
POST /api/github/repositories/rename
GET  /api/github/repositories/rename/jobs/{job_id}
POST /api/github/repositories/rename/rollback-preview
```

## 预览接口

```json
{
  "owner": "Chesterzero000",
  "current_name": "fishing-agent-robot",
  "target_name": "sustech-fishing-content-agent",
  "display_name_zh": "南科大钓鱼协会内容机器人",
  "reason": "仓库名需要体现南科大钓鱼协会和内容生产属性",
  "dry_run": true
}
```

返回内容应包含权限检查、slug 校验、目标名称可用性、旧 URL、新 URL、本地 remote 更新命令和部署/外部链接检查项。

## 执行接口

```json
{
  "owner": "Chesterzero000",
  "current_name": "fishing-agent-robot",
  "target_name": "sustech-fishing-content-agent",
  "display_name_zh": "南科大钓鱼协会内容机器人",
  "confirm": true,
  "expected_current_full_name": "Chesterzero000/fishing-agent-robot"
}
```

## 执行层映射

```http
PATCH https://api.github.com/repos/{owner}/{repo}
```

```json
{
  "name": "sustech-fishing-content-agent"
}
```

## 错误码

| 错误码 | HTTP | 含义 |
| --- | --- | --- |
| `REPO_NOT_FOUND` | 404 | 当前仓库不存在或无权限访问 |
| `NO_ADMIN_PERMISSION` | 403 | 当前用户没有 admin 权限 |
| `INVALID_TARGET_NAME` | 400 | 新仓库名不符合 slug 规则 |
| `TARGET_NAME_TAKEN` | 409 | 目标仓库名已被占用 |
| `CONFIRM_REQUIRED` | 400 | 缺少确认参数 |
| `EXPECTED_REPO_MISMATCH` | 409 | 当前仓库名与预期不一致 |
| `GITHUB_API_FAILED` | 502 | GitHub 接口失败 |

## slug 校验规则

```text
^[a-z0-9][a-z0-9-]{1,98}[a-z0-9]$
```

## 审计日志

```json
{
  "event": "repository.rename",
  "actor": "Chesterzero000",
  "old_name": "fishing-agent-robot",
  "new_name": "sustech-fishing-content-agent",
  "display_name_zh": "南科大钓鱼协会内容机器人",
  "status": "completed"
}
```

## 前端交互

- 先点击 `预览影响`。
- 展示旧 URL、新 URL、本地 remote 命令和检查清单。
- 用户勾选确认后才能点击 `确认重命名`。

## 重命名后本地命令

```bash
git remote set-url origin https://github.com/Chesterzero000/sustech-fishing-content-agent.git
git remote -v
```
